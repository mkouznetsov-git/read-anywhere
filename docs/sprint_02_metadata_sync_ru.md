# Sprint 2: интернет-синхронизация metadata

## Что реализовано

Добавлена первая рабочая версия синхронизации metadata через собственный relay:

- `accountId` и `deviceId` сохраняются локально в `manifest.json`;
- экран **Синхронизация** позволяет указать relay URL;
- manual pairing MVP: на двух устройствах нужно вручную указать одинаковый `accountId`;
- при подключении устройство отправляет `library_snapshot`;
- при добавлении книги, обновлении прогресса и добавлении закладки приложение отправляет новый snapshot;
- входящий snapshot сливается с локальной библиотекой через `mergeManifests`;
- локальный путь к файлу книги никогда не принимается от другого устройства;
- статус доступности книги хранится через `availableOnDeviceIds`.

## Что пока не реализовано

Это не финальная безопасная синхронизация. В этом спринте пока нет:

- QR-pairing;
- обмена публичными ключами;
- end-to-end шифрования сообщений;
- подписи событий;
- offline queue на relay;
- передачи самих файлов книг;
- автоматического скачивания книги с другого устройства.

Иными словами, текущий sprint нужен, чтобы проверить базовый сценарий: **Mac и Android через интернет видят одну библиотеку, прогресс и закладки**.

## Как запустить relay

```bash
cd server/rendezvous_relay
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8787
```

Проверка:

```bash
curl http://127.0.0.1:8787/health
```

Для теста через интернет relay должен быть доступен обоим устройствам. Подойдут:

- VPS с открытым портом `8787`;
- домашний сервер с port forwarding;
- временный туннель к локальному relay;
- локальная сеть для первого smoke-test, но целевая архитектура рассчитана на интернет.

## Как проверить Mac ↔ Android

### 1. Запустите relay

Например:

```bash
uvicorn main:app --host 0.0.0.0 --port 8787
```

Пусть публичный адрес relay будет:

```text
http://YOUR_SERVER_IP:8787
```

### 2. Установите приложения

Соберите через GitHub Actions:

- `ReadAnywhere-android-apk` для Android;
- `ReadAnywhere-macos-dmg-pkg` для macOS.

### 3. Настройте первое устройство

На Mac:

1. Откройте приложение.
2. Нажмите иконку синхронизации.
3. В поле `Relay URL` введите `http://YOUR_SERVER_IP:8787`.
4. Скопируйте `accountId`.
5. Нажмите **Подключиться**.
6. Вернитесь в библиотеку и добавьте TXT-книгу.

### 4. Настройте второе устройство

На Android:

1. Откройте приложение.
2. Нажмите иконку синхронизации.
3. Вставьте тот же `accountId`, который был на Mac.
4. Укажите свой `Relay URL`.
5. Нажмите **Сохранить**.
6. Нажмите **Подключиться**.
7. Нажмите **Отправить snapshot библиотеки** на Mac или Android, если snapshot не пришёл сразу.

### 5. Ожидаемый результат

На Android должна появиться книга, добавленная на Mac, со статусом:

```text
Не скачана здесь • доступна на 1 устройстве(ах)
```

Если открыть TXT-книгу на устройстве, где она скачана, прогресс чтения должен отправиться второму устройству после прокрутки.

## Правила merge

### Книги

Книги объединяются по `book.id`, где `id` — SHA-256 содержимого файла.

### Локальные файлы

`localPath` никогда не принимается от другого устройства. Это важно, потому что путь `/Users/alice/...` с Mac бессмысленен и небезопасен на Android.

### Доступность файла

Вместо передачи `localPath` между устройствами используется поле:

```json
"availableOnDeviceIds": ["device-a", "device-b"]
```

Оно показывает, на каких устройствах есть полный оригинальный файл книги.

### Прогресс чтения

Побеждает запись с более новым `progressVersion`. Если версии равны, используется `updatedAt`, затем `updatedByDeviceId` как tie-breaker.

### Закладки

Закладки объединяются по `bookmark.id`. Удалённые закладки в будущем будут передаваться как tombstone через `deletedAt`.

## Android и HTTP relay

Для удобства MVP скрипт `scripts/prepare_flutter_platforms.sh` добавляет Android `INTERNET` permission и временно включает `android:usesCleartextTraffic="true"`, чтобы debug APK мог подключаться к `ws://` / `http://` relay.

Для production это нужно заменить на HTTPS/WSS и убрать cleartext traffic.
