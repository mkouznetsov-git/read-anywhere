# Sprint 3 Hotfix 02 — восстановление metadata sync

## Проблема

После добавления передачи файлов в Sprint 3 у части сценариев перестала стабильно синхронизироваться библиотека:

- оба клиента показывали подключение к relay;
- локально книга импортировалась;
- на другом устройстве библиотека оставалась пустой;
- TXT-reader был исправлен в Hotfix 01, но metadata sync всё ещё зависел от порядка доставки событий.

## Причина

Клиентская синхронизация была слишком чувствительна к гонкам на старте подключения:

1. старый relay отправлял `peer_joined` только уже подключенным устройствам;
2. новое устройство не получало список существующих peers;
3. если `library_snapshot_requested` или первый `library_snapshot` уходил слишком рано после подключения, второй клиент мог не получить актуальный snapshot;
4. relay не имел возможности помочь новому клиенту восстановить последнее состояние metadata.

## Исправление

### Relay 0.1.1

Relay теперь:

- хранит список online-устройств по accountId;
- отправляет новому клиенту `peer_list` сразу после подключения;
- держит последние `library_snapshot` каждого устройства только в памяти процесса;
- отдаёт cached snapshots новому подключившемуся устройству;
- отдаёт cached snapshots в ответ на `library_snapshot_requested`;
- по-прежнему не хранит книги и не пишет данные на диск;
- не кэширует `book_file_chunk` и другие file-transfer события.

Кэш metadata исчезает при перезапуске relay.

### Клиент

Клиент теперь:

- понимает системное событие `peer_list`;
- после подключения делает `request + snapshot`;
- повторяет metadata-refresh через 600 мс, 2 секунды и 5 секунд после подключения;
- при `peer_joined` делает полный metadata-refresh;
- сохраняет совместимость с file-transfer Sprint 3.

## Проверка

1. Обновить relay на версию из этого hotfix.
2. Перезапустить relay:

```bash
cd server/rendezvous_relay
source .venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8787
```

3. Проверить health:

```bash
curl http://SERVER_IP:8787/health
```

После подключения двух устройств ответ должен содержать одну комнату и два device id.

4. Собрать и установить новые клиенты.
5. На Mac подключиться к relay и отправить snapshot библиотеки.
6. На Android подключиться к тому же accountId.
7. Android должен получить библиотеку автоматически. Если Android был подключён раньше Mac, подождать несколько секунд или нажать "Запросить snapshot".

## Ограничение MVP

Metadata snapshot в relay пока plaintext и хранится в RAM. Для production следующий шаг — E2E-шифрование snapshot/events и QR-pairing с ключами устройств.
