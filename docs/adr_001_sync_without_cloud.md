# ADR-001: Синхронизация без облачного хранения

## Status

Proposed

## Context

Требование: не использовать Google Drive, Яндекс Диск и другие сторонние облачные хранилища. Книги должны полностью копироваться на устройства аккаунта, выбранные пользователем. Метаданные библиотеки, закладки и прогресс должны быть видны на всех устройствах.

## Decision

Используем local-first архитектуру:

- source of truth — локальные устройства пользователя;
- metadata синхронизируется между устройствами;
- файлы книг передаются peer-to-peer и только по выбору пользователя;
- relay допускается только как самохостируемый временный транспорт без persistent storage;
- весь payload relay шифруется end-to-end.

## Consequences

Плюсы:

- нет облачного хранения книг;
- пользователь владеет данными;
- приложение работает offline;
- библиотека переживает отключение серверов.

Минусы:

- если ни одно устройство с нужной книгой не онлайн, новое устройство не скачает файл;
- pairing и NAT traversal сложнее, чем обычная облачная синхронизация;
- iOS/Android ограничивают долгую фоновую передачу файлов.

## Open questions

- Нужен ли официальный hosted relay от разработчика, если он не хранит данные?
- Должна ли быть поддержка семейного аккаунта с несколькими профилями чтения?
- Нужна ли поддержка DRM/LCP?
