# ADR-003: Metadata sync MVP через relay snapshots

## Статус

Accepted for MVP.

## Контекст

Для проверки продуктовой идеи нужно быстро получить сценарий, в котором устройства через интернет видят общую библиотеку, прогресс чтения и закладки. Полный P2P-файлообмен, QR-pairing и E2E encryption требуют отдельного спринта.

## Решение

На этапе MVP устройства подключаются к self-hosted WebSocket relay по комнате `accountId`.

Устройство отправляет событие:

```json
{
  "type": "library_snapshot",
  "accountId": "account-...",
  "deviceId": "device-...",
  "payload": {
    "reason": "book_imported",
    "manifest": {}
  }
}
```

Relay не сохраняет сообщения и не хранит книги. Он только пересылает JSON между подключенными устройствами одного `accountId`.

## Последствия

Плюсы:

- быстро проверяется интернет-синхронизация;
- не используется стороннее облачное хранилище;
- сервер не хранит книги;
- устройства остаются источниками истины для файлов.

Минусы:

- ручной `accountId` не является безопасным pairing;
- offline-доставка пока отсутствует;
- relay видит незашифрованные metadata-сообщения;
- snapshot-подход проще, но менее эффективен, чем event log/CRDT.

## Дальнейшее развитие

Следующая версия должна заменить ручной `accountId` на QR-pairing, добавить ключи устройств, подпись событий и E2E encryption. После этого можно добавить encrypted pending queue на relay и передачу файлов книг chunk-by-chunk.
